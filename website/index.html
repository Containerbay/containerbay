Containerbay

Web gateway for OCI artifacts

Hello from a container image !

This website is served by Containerbay but the content is hosted on quay.io as a standard OCI image artifact!

Containerbay is a container image proxy.

Some examples to browse:
  • Browse openSUSE/leap from DockerHub
  • Browse alpine from DockerHub
  • this website
You can use it in three ways:
  • By using the API E.g. to list opensuse/leap:latest container directory:
    curl https://containerbay.io/docker.io/opensuse/leap/etc/os-release
  • By using the containerbay magicdns:
    curl http://docker.io.library.alpine.latest.containerbay.io/etc/os-release or for example our website from here
  • With a custom domain: You can set up containerbay.io as CNAME for your own domain and a corresponding TXT DNS record which tells containerbay which container image to serve.
    Set a CNAME record pointing to containerbay.io and a TXT DNS record for the corresponding domain you want to serve with the following content:
    containerbay=docker.io/library/alpine
    It is strongly adviced to use a custom registry which doesn't limit fetching image manifest digests.
You can also deploy Containerbay on your own hardware, check the Project on Github
